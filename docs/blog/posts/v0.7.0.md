# grok_switch v0.7.0：内置注册机、CPA 铸造与认证目录热加载

v0.7.0 在号池之上补齐账号获取链路：内置注册机（Cloudflare / CloudMail / Hotmail）、本机 CPA 设备授权铸造，以及统一的 CPA 认证目录导入与热加载。铸造与注册产出可直接进入号池巡检，默认路径按当前用户主目录解析，不绑定固定机器路径。

<!-- more -->

## 内置注册机

管理中心新增「注册机」面板，支持：

- **Cloudflare 临时邮**（默认）：配置 API Base 与可选鉴权即可批量注册
- **CloudMail**：管理员邮箱 + 域名
- **Hotmail 四段凭证**：`邮箱----密码----ClientID----refresh_token`

任务支持进度、日志尾部与停止；探测（Probe）可在启动前检查浏览器与认证目录。配置保存在本机数据目录，权限收紧。

## CPA 设备授权铸造

号池侧提供 CPA 设备授权流程：

1. Go 服务申请 device code 并轮询 token
2. 系统浏览器完成登录 / Turnstile / 允许
3. 写出 CPA 兼容 `xai-*.json` 并自动导入号池

此流程用于为已有 xAI 账号铸造 CLI 凭据，不会在本机创建新账号。

## 统一 CPA 认证目录

号池设置中的「CPA 认证目录」同时用于：

- 从认证目录导入
- 目录热加载（监视新 JSON）
- CPA 铸造与注册机输出

默认路径为 `%USERPROFILE%\.grok_switch\cpa_auths`（macOS/Linux 为 `~/.grok_switch/cpa_auths`），也可在 UI 中改为任意本机路径。

## 下载与校验

本版本同时提供：

- `grok_switch.exe`（托盘 + 浏览器面板）
- `grok_switch_gui.exe`（Wails 桌面 GUI）
- 各自对应的 `.sha256` 校验文件

> 本版本未配置 Authenticode 证书，发布产物为未签名 Windows 构建。首次运行可能出现 SmartScreen 提示，可用 SHA-256 文件核对下载内容。

## 升级提示

1. 先退出旧版托盘 / GUI 进程，再替换为 v0.7.0 的 exe。
2. 配置、号池、备份与 Grok 本地会话仍保存在用户目录；新注册相关数据写入 `~/.grok_switch/registrar*` 与 CPA 目录。
3. 注册机依赖本机 Chrome/Edge（chromedp）；Hotmail / CloudMail 需自行准备邮箱侧凭据。
4. 若浏览器仍缓存旧前端，请强制刷新（Ctrl+F5）；使用内嵌 UI 的 exe 时以本版本重新编译产物为准。

# grok_switch v0.6.1：聊天正文修复、iOS 气泡与对话体验补强

v0.6.1 聚焦聊天页体验。修复了用户与助手气泡只显示角色名、看不到正文的问题；消息外观改为 iOS Messages 风格气泡；并补齐附件发送、消息编辑重发、停止生成、长历史增量加载、会话内查找与节点跳转等常用能力。Agent 侧同步支持附件入 prompt、按轮次取消生成，以及本会话自动批准工具。

<!-- more -->

## 修复：聊天正文不再空白

此前用户气泡和助手气泡有时只显示「你」或「Grok」，正文区域为空。历史消息与新建消息都会受影响。

根因是消息节点在挂入 DOM **之前**就调用了 Markdown 渲染，而渲染函数在 `!isConnected` 时直接返回，正文从未写入。v0.6.1 调整为：

1. 先把消息挂到对话列表，再渲染 Markdown / 附件；
2. 正文写入不再依赖 `isConnected`（Mermaid 仍要求节点已在文档中以便布局）；
3. 使用渲染代数戳，避免流式更新与历史批量渲染互相覆盖。

升级后重新打开会话或发送新消息，应能看到完整用户提问与 Grok 回复。

## iOS 风格消息气泡

用户与助手消息恢复气泡布局，视觉参考 iOS 信息：

| | 用户 | 助手 |
|--|------|------|
| 对齐 | 右对齐 | 左对齐 |
| 颜色 | iOS 蓝 `#007AFF`，白字 | 浅灰 `#E9E9EB`，深字 |
| 圆角 | 大圆角，右下略尖 | 大圆角，左下略尖 |

气泡内 Markdown、代码块、链接与表格做了对比度适配；系统提示仍为左侧提示条，不使用聊天气泡。自定义聊天背景时，气泡会保留轻阴影，助手气泡可带轻微毛玻璃。

## 对话能力补强

聊天页新增或完善了这些日常能力：

- **附件**：支持图片与文本文件随消息发送；Agent 会将图片与文本片段一并写入 ACP prompt。
- **编辑并重发**：可编辑已发送的用户消息，并截断其后 UI 后重新提交。
- **重新生成**：对最后一条助手回复一键重跑。
- **停止生成**：生成中可取消当前轮次（后端支持按 turn 取消）。
- **长历史增量加载**：历史会话按窗口渲染，可向上加载更早消息，减轻超长会话卡顿。
- **会话内查找**：在当前对话中搜索消息内容。
- **节点导航**：按用户消息生成可点击节点，快速跳转到对应轮次。
- **上下文用量粗估**：按字符量估算 token，便于感知对话体量。
- **会话重命名**：为历史会话设置自定义标题并持久化。
- **恢复上次对话上下文**：重新进入聊天页时尽量恢复最近会话与工作目录。

## Agent 与权限

- 本会话可开启 **自动批准工具**（`session_auto_approve`），与全局 always-approve 区分。
- 工具权限请求、重试状态与流式回复仍通过本机 WebSocket 转发，不直接暴露 Grok Agent 端口。

## 下载与校验

本版本同时提供：

- `grok_switch.exe`（托盘 + 浏览器面板）
- `grok_switch_gui.exe`（Wails 桌面 GUI）
- 各自对应的 `.sha256` 校验文件

> 本版本未配置 Authenticode 证书，发布产物为未签名 Windows 构建。首次运行可能出现 SmartScreen 提示，可用 SHA-256 文件核对下载内容。

## 升级提示

1. 先退出旧版托盘 / GUI 进程，再替换为 v0.6.1 的 exe。
2. 供应商、账号池、设置、配置备份与 Grok 本地会话仍保存在用户目录，无需重新导入。
3. 聊天功能仍要求本机已安装并可执行 `grok`（Grok Build）。
4. 若浏览器仍缓存旧前端，请强制刷新（Ctrl+F5）；使用内嵌 UI 的 exe 时以本版本重新编译产物为准。
